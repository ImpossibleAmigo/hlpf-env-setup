# Практична робота №8: Фінальний проєкт MiniShop

**Виконав:** Чеберяко Олексій
**Група:** Спеціальність 121 (3-й курс)

## Опис роботи
У межах фінального проєкту реалізовано модуль замовлень (`OrdersModule`), який об'єднує всі вивчені технології:
1. **Транзакційність**: Створення замовлення відбувається в одній транзакції. Якщо товару недостатньо або виникає помилка, `stock` продукту не змінюється.
2. **Ownership Check**: Користувачі мають доступ лише до власних замовлень (реалізовано через перевірку `userId` у сервісі).
3. **RBAC (Role Based Access Control)**: Зміна статусу та видалення замовлень доступні лише для користувачів із роллю `admin`.
4. **Валідація**: Використано `ValidateNested` для перевірки масиву товарів у замовленні.

## API Endpoints
| Метод | Шлях | Опис | Доступ |
|---|---|---|---|
| POST | `/api/orders` | Створення замовлення | User |
| GET | `/api/orders` | Список замовлень | User (свої) / Admin (всі) |
| GET | `/api/orders/:id` | Деталі замовлення | Owner / Admin |
| PATCH | `/api/orders/:id/status` | Оновлення статусу | Admin |
| DELETE | `/api/orders/:id` | Видалення замовлення | Admin |

## Результати тестування
- **Успішне створення**: Замовлення створюється, `totalPrice` рахується автоматично.
- **Stock Control**: При замовленні 2-х одиниць iPhone, залишок у базі зменшується відповідно.
- **Security**: При спробі користувача Bob отримати доступ до замовлення Alice, API повертає `403 Forbidden`.
